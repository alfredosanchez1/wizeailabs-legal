3. Datos Biométricos y Autenticación

Uso de Face ID / Touch ID

RecipeTuner utiliza Face ID y Touch ID exclusivamente para autenticación local del dispositivo, proporcionando acceso seguro y conveniente a tu cuenta.

Aclaraciones Importantes

NO recopilamos, procesamos ni almacenamos ningún dato biométrico:

  • ✅ Sin Recopilación: No capturamos ni recopilamos datos de reconocimiento facial, huellas dactilares o cualquier otra información biométrica
  • ✅ iOS lo Maneja Todo: Face ID y Touch ID son manejados completamente por iOS en el Secure Enclave de tu dispositivo (un chip de seguridad dedicado)
  • ✅ Solo Recibimos Éxito/Fallo: Nuestra app solo recibe una respuesta simple de "autenticación exitosa" o "autenticación fallida" de iOS
  • ✅ Los Datos Nunca Salen de Tu Dispositivo: Ninguna información biométrica sale de tu dispositivo ni se transmite a nuestros servidores
  • ✅ No se Comparte con Nadie: No compartimos datos biométricos con terceros porque nunca tenemos acceso a ellos

Cómo Funciona

Cuando activas Face ID/Touch ID en RecipeTuner:

  1. Tocas "Desbloquear con Face ID"
  2. iOS te solicita autenticarte usando tus datos biométricos registrados
  3. iOS procesa la autenticación completamente dentro del Secure Enclave
  4. iOS le informa a RecipeTuner solo si la autenticación fue exitosa o falló
  5. Si es exitosa, obtienes acceso a la app

En ningún momento RecipeTuner accede, ve o almacena tus datos biométricos.

Ubicación de Almacenamiento de Datos

Almacenamiento de datos biométricos: Secure Enclave de tu dispositivo (gestionado por Apple)

Almacenamiento de RecipeTuner: Ninguno

Todos los datos biométricos (patrones de reconocimiento facial, plantillas de huellas dactilares) se almacenan exclusivamente en el Secure Enclave de tu dispositivo, que es:

  • Encriptado por hardware
  • Aislado del procesador principal
  • Inaccesible para todas las aplicaciones (incluido RecipeTuner)
  • Gestionado completamente por el sistema operativo de Apple

Control del Usuario

Tienes control completo sobre la autenticación biométrica:

Activar/Desactivar: Puedes activar o desactivar Face ID/Touch ID para RecipeTuner en cualquier momento en:

  • Ajustes de iOS → RecipeTuner → Face ID y Código, o
  • Ajustes de la app RecipeTuner → Seguridad

Autenticación Alternativa: Siempre puedes usar tu correo/contraseña para iniciar sesión en lugar de autenticación biométrica

Eliminar Datos Biométricos: Para eliminar tus datos biométricos, ve a Ajustes de iOS → Face ID y Código → Restablecer Face ID (gestionado por Apple, no por RecipeTuner)

Acceso de Terceros

Ninguno.

Apple no comparte datos biométricos con desarrolladores de apps o terceros. Tus datos de Face ID/Touch ID están protegidos por la arquitectura de seguridad de Apple y nunca son accesibles para RecipeTuner o cualquier otra aplicación.

Retención de Datos

Retención de RecipeTuner: No aplica - no recopilamos datos biométricos

Retención de Apple: Apple gestiona toda la retención de datos biométricos según sus políticas de privacidad. Cuando eliminas Face ID de tu dispositivo, Apple elimina las plantillas biométricas.

Implementación Técnica

RecipeTuner utiliza la API LocalAuthentication de Expo, que es un wrapper seguro del framework LocalAuthentication de Apple. Esto garantiza:

  • Prácticas de seguridad estándar de la industria
  • Sin acceso directo a datos biométricos
  • Cumplimiento con las guías de seguridad de Apple
  • Protección de tu privacidad

Cumplimiento

Nuestro uso de autenticación biométrica cumple con:

  • Guías de Revisión de App Store de Apple (Guideline 2.5.13)
  • Requisitos del GDPR (sin procesamiento de datos biométricos)
  • Requisitos de CCPA (sin recopilación de datos biométricos)
  • Guías de Seguridad de la Plataforma Apple

¿Preguntas sobre Datos Biométricos?

Si tienes preguntas sobre cómo Apple maneja Face ID/Touch ID, consulta:

  • Guía de Seguridad de Face ID de Apple
  • Política de Privacidad de Apple

Para preguntas sobre las prácticas de seguridad de RecipeTuner, contáctanos en: privacy@wizeailabs.com

3. Biometric Data & Authentication

Face ID / Touch ID Usage

RecipeTuner uses Face ID and Touch ID exclusively for local device authentication to provide secure and convenient access to your account.

Important Clarifications

We DO NOT collect, process, or store any biometric data:

  • ✅ No Collection: We do not capture or collect facial recognition data, fingerprints, or any other biometric information
  • ✅ iOS Handles Everything: Face ID and Touch ID are handled entirely by iOS in your device's Secure Enclave (a dedicated security chip)
  • ✅ We Only Receive Success/Failure: Our app only receives a simple "authentication successful" or "authentication failed" response from iOS
  • ✅ Data Never Leaves Your Device: No biometric information ever leaves your device or is transmitted to our servers
  • ✅ Not Shared With Anyone: We do not share biometric data with third parties because we never have access to it

How It Works

When you enable Face ID/Touch ID in RecipeTuner:

  1. You tap "Unlock with Face ID"
  2. iOS prompts you to authenticate using your registered biometric data
  3. iOS processes the authentication entirely within the Secure Enclave
  4. iOS tells RecipeTuner only whether authentication succeeded or failed
  5. If successful, you gain access to the app

At no point does RecipeTuner access, see, or store your biometric data.

Data Storage Location

Biometric data storage: Your device's Secure Enclave (managed by Apple)

RecipeTuner storage: None

All biometric data (facial recognition patterns, fingerprint templates) is stored exclusively in your device's Secure Enclave, which is:

  • Hardware-encrypted
  • Isolated from the main processor
  • Inaccessible to all applications (including RecipeTuner)
  • Managed entirely by Apple's operating system

User Control

You have complete control over biometric authentication:

Enable/Disable: You can enable or disable Face ID/Touch ID for RecipeTuner at any time in:

  • iOS Settings → RecipeTuner → Face ID & Passcode, or
  • RecipeTuner app Settings → Security

Alternative Authentication: You can always use your email/password to log in instead of biometric authentication

Delete Biometric Data: To delete your biometric data, go to iOS Settings → Face ID & Passcode → Reset Face ID (managed by Apple, not RecipeTuner)

Third-Party Access

None.

Apple does not share biometric data with app developers or third parties. Your Face ID/Touch ID data is protected by Apple's security architecture and is never accessible to RecipeTuner or any other application.

Data Retention

RecipeTuner retention: Not applicable - we do not collect biometric data

Apple's retention: Apple manages all biometric data retention according to their privacy policies. When you delete Face ID from your device, Apple deletes the biometric templates.

Technical Implementation

RecipeTuner uses the Expo LocalAuthentication API, which is a secure wrapper around Apple's LocalAuthentication framework. This ensures:

  • Industry-standard security practices
  • No direct access to biometric data
  • Compliance with Apple's security guidelines
  • Protection of your privacy

Compliance

Our use of biometric authentication complies with:

  • Apple App Store Review Guidelines (Guideline 2.5.13)
  • GDPR requirements (no biometric data processing)
  • CCPA requirements (no biometric data collection)
  • Apple Platform Security Guidelines

Questions About Biometric Data?
